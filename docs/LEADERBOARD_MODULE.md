## Overview

The Leaderboard Module is responsible for displaying tournament rankings, contestant standings, and statistics. It consumes ranking data generated by the Scoring Engine and presents it in a rich, responsive, and user-friendly interface.

**Key Principle:** The Leaderboard module does NOT calculate scores. It only displays pre-calculated rankings from the Scoring Engine.

## Features

### Contestant View

- **Tournament Leaderboard**: View current rankings with points, accuracy, and statistics
- **My Statistics**: Personal performance metrics and ranking
- **Search**: Find contestants by name or country
- **Filters**: View top 10/25/50, or contestants around your position
- **Responsive Design**: Automatic table/card switching based on device

### Administrator View

- **All Tournaments**: Switch between tournament leaderboards
- **Refresh**: Manually refresh leaderboard data
- **Visibility Toggle**: Control leaderboard visibility (structure prepared)
- **Full Statistics**: Tournament-wide metrics and completion rates

## Data Model

### Leaderboard Entry

```javascript
{
  userId: string,
  rank: number,
  totalPoints: number,
  correctWinnerCount: number,
  exactScoreCount: number,
  bonusPoints: number,
  accuracy: number,              // Percentage
  matchesPredicted: number,
  matchesRemaining: number,
  previousRank: number | null,
  movement: 'up' | 'down' | 'same' | 'new',
  displayName: string,
  photoURL: string | null,
  country: string | null,
  isCurrentUser: boolean
}
```

### Leaderboard Data Source

The module reads from:

1. **`leaderboard_cache` collection**: Stores user points totals per tournament
   ```javascript
   {
     tournamentId: string,
     matchId: string,
     totals: { [userId]: points },
     updatedAt: Timestamp
   }
   ```

2. **`predictions` collection**: For detailed statistics (correct winners, exact scores)

3. **`users` collection**: For contestant display names, photos, countries

4. **`matches` collection**: For match counts and completion status

## Ranking Logic

### Ranking Rules (Applied by Domain Layer)

1. **Primary Sort**: Total points (descending)
2. **Tie-Breaker**: Configured per tournament via `configuration.tieBreaker`
   - Default: `displayName` (alphabetical)
   - Configurable: `correctWinnerPredictions`, `exactScorePredictions`, etc.

### Movement Indicators

- **↑ (Up)**: Rank improved since last update
- **↓ (Down)**: Rank decreased since last update
- **→ (Same)**: Rank unchanged
- **NEW**: First appearance in leaderboard

*Note: Movement tracking requires historical rank storage (future enhancement)*

## Leaderboard Visibility

Controlled by platform settings: `settings/general.leaderboardVisible` (managed at `/admin/settings`)

- **`true`**: Leaderboard visible to all contestants
- **`false`**: Leaderboard hidden (only admins can view)

The `leaderboard.guard.js` enforces this at the route level.

## Responsive Design

### Desktop (≥768px)
- Table layout with sticky header
- All columns visible (rank, points, winners, exact scores, accuracy, predicted, movement)
- Hover effects and highlighting for current user

### Mobile (<768px)
- Card-based layout
- Compact information display
- Touch-friendly interface
- Essential stats prominently displayed

## Caching Strategy

- **Cache Duration**: 5 minutes
- **Cache Key**: `{tournamentId}:{currentUserId}`
- **Manual Refresh**: Available via "Refresh" button
- **Cache Invalidation**: On scoring completion events (future enhancement)

## Search and Filtering

### Search
- Search by contestant name (case-insensitive)
- Search by country (case-insensitive)
- Debounced input (300ms delay)

### Filters
- **All**: Show all contestants
- **Top 10**: Show top 10 ranks
- **Top 25**: Show top 25 ranks
- **Top 50**: Show top 50 ranks
- **My Position**: Show contestants within ±5 ranks of current user

## Statistics

### Contestant Statistics
- Current rank and movement
- Total points
- Correct winner predictions
- Exact score predictions
- Bonus points
- Prediction accuracy (%)
- Predictions submitted/remaining

### Tournament Statistics
- Total contestants

- Total predictions
- Prediction completion rate (%)
- Average accuracy (%)

## Events

### Emitted Events
- `leaderboard:loaded`: When leaderboard data is loaded
- `leaderboard:error`: When an error occurs
- `leaderboard:refreshed`: When leaderboard is manually refreshed
- `leaderboard:filter_changed`: When filter is changed
- `leaderboard:sort_changed`: When sort option is changed
- `leaderboard:search_changed`: When search term is updated

## Security

### Firestore Rules

```javascript
// Read leaderboard_cache
match /leaderboard_cache/{tournamentId} {
  allow read: if request.auth != null;
}

// Predictions and users follow existing security rules
```

### Permission Checks
- Contestants: Can only view if platform `leaderboardVisible === true`
- Admins: Always have access regardless of visibility setting
